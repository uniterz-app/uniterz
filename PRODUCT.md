# Product

## Register

product

## Users

ガチ勢のスポーツ予想ユーザー。NBA・FIFA ワールドカップ・J リーグなどの試合スコアを予想し、的中精度・連勝・アップセット的中などの指標で他ユーザーと競う。試合前後にモバイル中心で利用し、ランキングやリザルトで自分の成績を分析することに価値を感じる。多言語対応（ja/en/es/fr/de/pt/ko/zh/ar）済みでグローバル展開も視野。

主要画面: 予想フォーム、リザルトカード、ランキング（NBA / WC ステージ別）、プロフィール、コミュニティ、ブラケット。Web（`/web`）とモバイル（`/mobile`）の 2 レイアウト + React Native アプリ。

## Product Purpose

Uniterz — スポーツ試合予想プラットフォーム。スコア予想の精度を多面的な指標（総合スコア・スコア精度・アップセット・連勝・得点者的中）でスコアリングし、競技性のあるランキングで「予想のうまさ」を可視化する。成功 = ユーザーが試合のたびに戻ってきて予想し、自分の精度の伸びとランキング順位に熱中している状態。

## Brand Personality

サイバー・競技的・熱狂。e スポーツの精密な UI（Valorant / Riot 系の HUD 的正確さ）が理想の雰囲気。ダーク背景（#081116 系）にシアン/ネオンのアクセント、Oxanium / Rajdhani / Bebas Neue などのディスプレイフォントで「競技の場」を演出する。現在の方向性を磨き込む（置き換えない）。

## Anti-references

- ギャンブル / ブックメーカー的な安っぽさ（金と赤の煽り、オッズ煽り UI）
- 汎用 SaaS の無個性（白背景、紫グラデーション、テンプレ感）
- 情報詰め込みすぎのゴチャゴチャしたダッシュボード（読めない数字の羅列）

## Design Principles

1. **競技の場の精密さ** — e スポーツ HUD のように、数字と階層が一目で読める。装飾は情報の邪魔をしない。
2. **数字が主役** — スコア・順位・精度が最初に目に入る。タイポグラフィの階層で勝敗と成績を語る。
3. **熱狂は演出で、構造は冷静に** — スイープやグローは勝利・連勝など「瞬間」に限定。恒常的な UI は静かに保つ。
4. **モバイルファースト** — 試合前後のスマホ利用が主戦場。タップ領域・親指圏・縦スクロールを最優先。
5. **既存システムを尊重** — 既存のトークン・フォント・カラーを基準に磨く。新しい見た目を発明しない。

## Accessibility & Inclusion

- WCAG AA を明示的に満たす（本文コントラスト ≥4.5:1、大きい文字 ≥3:1）
- `prefers-reduced-motion` 対応は全アニメーションで必須（既存のスイープ演出は対応済み、新規も同様に）
- 多言語 UI（RTL の ar を含む）でレイアウトが破綻しないこと
- ダーク UI 上のシアン/ネオン文字は実コントラストを必ず検証する
